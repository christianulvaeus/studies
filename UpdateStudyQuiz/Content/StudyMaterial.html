Vilka systemdefinierade User Defined Routes finns i ett Azure VNet? Beskriv kortfattat varje.

  • Local Vnet Rule: This rule is automatically created for every subnet in a virtual network. It specifies that there is a direct link between the VMs in the VNet and there is no intermediate next hop.
  • On-premises Rule: This rule applies to all traffic destined to the on-premises address range and uses VPN gateway as the next hop destination.
  • Internet Rule: This rule handles all traffic destined to the public Internet (address prefix 0.0.0.0/0) and uses the infrastructure internet gateway as the next hop for all traffic destined to the Internet.

Vilka alternativ finns för Redundant Storage i Azure Storage Services?

Varför använda ett VNet och hur?

  • Ett VNet används för att begränsa kommunikation mellan VM/tjänster inuti nätet och utanför. Man anger giltiga IP adresser i nätverket. Man kombinerar en start-IP med en nätverksmask och får då en range. En syntax för sådana definitioner är CIDR.

Hurdan syntax för CIDR notation och ge exempel på masker.

  • CIDR specifies an address range using a combination of an IP address and its associated network mask. CIDR notation uses this format: xxx.xxx.xxx.xxx/n, where n is the number of leftmost “1” bits in the mask. For example, 192.168.12.0/23 applies the network mask 255.255.254.0 to the 192.168 network, starting at 192.168.12.0. This notation therefore represents the address range 192.168.12.0–192.168.13.255. Masken anger vilka bitar som får variera. Där masken har bit 1 är IP adressen fixd. Alltså t.ex. /22 betyder att inom 3:e segmentet (17 tom 24) är 23:e bit 21 och 24:e bit 20 “fria” och låter alltså detta segment variera med [0,3].

Varför använda ett Subnet i ett VNet och hur?

  • Subnets används för att dela in VNet i olika logiska undergrupperingar (t.ex. VMs i 10.1.0.0, SQL Server i 10.2.0.0 osv). Man kan också ange regler för Network Security Groups (NSGs) inom ett subnät, men standard är full kommunikation mellan subnät i ett VNet. Ett annat exempel för subnät är för tilldelning av fasta IP-adresser som, om dom låg i gemensamt nät, riskerade krocka med dynamiska systrar.

På vilka sätt kan ett VNet nås eller expanderas till lokala faciliteter?

  • Det finns Site-to-site vilket innebär nätverk-till-Azure virtuellt nätverk ("hela kontoret"). Detta kräver speciell VPN utrustning.
  • Man kan också använda Point-to-site som är från en specifik maskin. Kräver manuell installation på varje sådan maskin samt en manuellt/auto-startad VPN klient.
  • Finns också Private site-to-site (ExpressRoute) som kräver avancerad programvara/utrustning och speciell installation/konfiguration hos ISPn.

Jämför VNet site-to-site med point-to-site.

  • You don’t need a VPN device or RRAS to use a point-to-site network.
  • With point-to-site, configuration must be done on each client machine. With site-to-site, no changes are required to the client machines.
  • Point-to-site is a good choice when:
  • You only have a few clients that need to have access.
  • You don’t have access to a VPN device that you can use for a site-to-site connection.
  • You want to connect securely when off site (such as at a customer site or a coffee shop).
  • You can have both point-to-site and site-to-site networks running simultaneously. If you can create a site-to-site network, you might use site-to-site for people on premises but allow point-to-site for people who need to connect from a remote location.

Five minutes

Why if I have a backup frequency of 60 minutes in Azure Redis Cache and there is more than 60 minutes between backups?

  • The backup frequency interval does not start until the previous backup process has completed successfully.   • If you have scaled to a larger size, there is no impact.

  • If you have scaled to a smaller size, and there isn't enough room in the smaller size to hold all of the data from the last backup, keys will be evicted during the restore process, typically using the allkeys-lru eviction policy.
